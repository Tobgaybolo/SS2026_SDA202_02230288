# Reflection on Class Diagram and Object Diagram

The class diagram helped me understand the static structure of the Automated Grading System. It clearly outlines the main building blocks—such as Student, Professor, Submission, GradingSystem, Result, and MainframeLMS—and shows how they relate to one another. This gave me a high-level overview of the system’s components before considering specific data.

In contrast, the object diagram illustrated a real-world snapshot of the system at a specific point in time. Instead of abstract classes, it shows actual instances, such as a particular student’s submission, a specific grading result, and a concrete LMS sync. This made the design easier to grasp by walking through a tangible scenario, from submission to final grade.

Together, the two diagrams offered complementary perspectives: the class diagram defined the general design, while the object diagram demonstrated how that design works in practice. Using both diagrams made the system easier to learn, explain, and communicate to others.
